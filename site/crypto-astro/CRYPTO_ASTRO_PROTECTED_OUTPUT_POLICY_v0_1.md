## Purpose

This artifact defines the boundary between the public Crypto-Astro proof surface and protected subscriber-grade outputs.

The current public Crypto-Astro page is a public-safe proof/context layer. It must not become a public alpha feed, public trading signal layer, public forecast layer, or public subscriber report archive.

## Current public layer

The public layer may show:

- public-safe market context
- low-resolution status
- static proof state
- closure and governance artifacts
- boundary notes
- delayed or synthetic demo examples
- static schema without active weights
- no-trading-signal / no-forecast copy

The public layer is not an alpha layer.

## Protected output layer

The following output classes must remain private, gated, delayed, manual, subscriber-only, or otherwise protected:

- subscriber reports
- asset-specific interpretation
- timing windows
- active A/E numeric scores
- active Φ Balance outputs
- active Φ Resonance outputs
- validation performance outputs
- formula weights
- hidden thresholds
- private synthesis
- ORION reasoning
- runtime-generated alpha

## Public not allowed

The public repo, public UI, public JSON, public API, static proof artifacts, and public pages must not expose:

- buy / sell / hold language
- entry / exit language
- price targets
- asset rankings by opportunity
- live alpha feeds
- public APIs for subscriber-grade output
- historical alpha datasets
- backtest tables showing exploitable edge
- formula weights
- hidden thresholds
- protected ORION/Frey synthesis
- private client context

## Subscriber / private delivery layer

The protected delivery layer may include:

- operator-reviewed synthesis
- asset-specific context
- evidence/source notes
- private research notes
- delayed or manual delivery
- controlled snapshot interpretation

Any delivery layer must preserve safety boundaries and must not present outputs as financial advice.

## Protected core

The following must remain protected:

- ORION reasoning
- formula weights
- validation datasets
- hidden thresholds
- runtime alpha generation
- private client context
- paid/manual synthesis logic
- future protected calculation core

C/T Temporal · Ephemerides Support may use public-safe language such as:

C/T Temporal must not expose:

## Governance

Public HTML changes require a scoped PR route.

Protected-output policy changes require PR review.

Formula execution, A/E numeric activation, runtime/backend/API, live adapters, TVL implementation, and subscriber-grade delivery each require separate scoped plans.

## Do-not-open gates

This artifact does not authorize:

- formula execution
- A/E numeric activation
- TVL implementation
- runtime/backend/API
- live adapters
- public trading signals
- forecasts
- price targets
- investment recommendations
- subscriber report publication
- public alpha feed creation

## Current decision

Crypto-Astro public surface remains proof/context/boundary only.

Protected outputs remain private or gated.

ORION core remains protected.
